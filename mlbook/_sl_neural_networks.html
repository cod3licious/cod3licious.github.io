Overview

Recap: Linear Models

Prediction is a linear combination of input features (and intercept \(b\)):

\[f(\mathbf{x}; \mathbf{w}) = b + \langle\mathbf{w},\mathbf{x}\rangle = b + \sum_{k=1}^d w_k \cdot x_k = \hat{y}\]

In the case of multiple outputs \(\mathbf{y}\):

\[f(\mathbf{x}; W) = \mathbf{x^\top}W = \mathbf{\hat{y}}\]

Tips & Tricks

How to get your network to learn something

  • scale your data (for classification tasks only inputs, for regression tasks also outputs or adapt bias at the last layer; StandardScaler is usually a good choice)

  • use sample weights for classification problems with unequal class distributions

  • NN are trained with gradient descent, which requires a good learning rate (not too small, otherwise nothing is learned, not too big, otherwise it spirals out of control):

  • sanity check: a linear network (i.e. FFNN with only 1 layer mapping directly from the inputs to the outputs) should achieve approximately the same performance as the corresponding linear model from sklearn

  • gradually make the network more complex until it can perfectly memorize a small training dataset

  • when selecting hyperparameters, always check if there is a clear trend towards an optimal setting; if the pattern seems random, initialize your network with different random seeds to see how robust it is

  • a learning rate scheduler or early stopping can improve the generalization performance

  • but often it is more important to train the network long enough. really long.

→ more tips for training NN: http://karpathy.github.io/2019/04/25/recipe/

Self-Supervised & Transfer Learning

Self-supervised learning: learn from unlabeled data by predicting missing values or what will happen next (e.g. masked words in a sentence, next image in a video) → build up general knowledge about how the world works, similar to what humans do. This technique is related to unsupervised learning in that the models don’t need labeled examples, but instead pseudo labels are generated from the data itself (e.g. the first 5 words of a sentence are the inputs, the next word is the target, however, from all the texts on the Internet we can generate basically infinitely many such training examples at no (human labeling) cost).

NLP Example: Neural Network Language Models (e.g. word2vec): given k words, predict the next word in the sentence
→ learn word embeddings that capture semantic & syntactic relationships (which is ignored in TF-IDF vectors, where each word dimension has the same distance to all other words):

→ these word embedding vectors can then be used as input to a RNN

Transfer Learning: use what you’ve learned before on a different task (possibly a self-supervised learning task) as a starting point for a new task, i.e., initialize the weights of your network with the weights of the network trained on the other task.

Typically, not all weights are copied, but only those from the earlier layers, where the network has learned some general principles that are not task related (e.g. observe how the first layer of the CNN has learned to detect edges, which seems like a relevant skill for pretty much all computer vision tasks). Often, using a pretrained network, i.e., initializing the weights of your network with those of a network trained on a related task, will give you a more robust solution and boost the performance if you only have a very small dataset and if the source task and target task are sufficiently related to each other. However, since when training a neural network you’re trying to find weights that minimize your error function by iteratively improving the weights starting with some initialization, if this initialization is unfavorable because it is very far away from a good minimum (further away than a random initialization), e.g., because you’ve initialized the weights with those of a network trained on an unrelated task, then this will actually hurt the performance, since your network first has to unlearn a lot of things from this unrelated task before it can learn your actual task. Therefore, only use transfer learning if the tasks are related enough. Pretraining a network on a self-supervised learning task (i.e. a task that is just about understanding the world in general, not solving a different kind of specific task) usually works quite well.
Even when not copying weights though, you can still transfer an architecture in general that has worked well on a similar problem as yours. This is often very helpful already and might in some cases even be more crucial than the actual weights themselves.

class MyNeuralNet(nn.Module):
    def __init__(self, n_in, n_hl1, n_hl2, n_out=10):
        super(MyNeuralNet, self).__init__()
        # initialize layers
        self.l1 = nn.Linear(n_in, n_hl1)
        self.l2 = nn.Linear(n_hl1, n_hl2)
        self.lout = nn.Linear(n_hl2, n_out)
    def forward(self, x):
        # apply layers in correct order
        h = F.relu(self.l1(x))       # 784 -> 512 [relu]
        h = F.relu(self.l2(h))       # 512 -> 256 [relu]
        y = F.softmax(self.lout(h))  # 256 -> 10 [softmax]
        return y
