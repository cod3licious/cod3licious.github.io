When the problem calls for it, I bring a strong machine learning background to design data-driven and AI-powered solutions as part of the bigger picture.

The Challenge

Even experienced software engineering teams struggle with recurring issues:
  • Developers start coding with an incomplete understanding of the problem, and the resulting software is far more complicated than it needs to be.
  • Features take three times longer than they should because the codebase has become a maze of workarounds and accidental complexity.
  • The code is disconnected from the domain, so when a business process changes, no one knows what logic needs updating.
  • AI coding assistants and agents are changing how software gets built, but the team isn't set up to use them effectively, or the codebase isn't structured in a way that lets AI tools help rather than hallucinate.
  • Machine Learning (ML) and data-driven features get bolted on as an afterthought instead of being designed as a coherent part of the product.
  • The software wasn't designed for a future where AI agents interact with it: rigid GUIs, opaque file formats, and missing validation layers make it hard to automate and integrate.
The root cause is almost always the same: the problem wasn't deeply understood before the solution was designed.

A Better Way

I help your team avoid these pitfalls by bridging the gap between a complex domain and clean, maintainable software:

  • Deeply understand your domain, whether that's chemical processes, industrial systems, or another technically demanding field.
  • Translate that understanding into a clear software design that captures the essential complexity and nothing more.
  • Include ML and data-driven components where they genuinely add value — as part of the whole, not a standalone prototype.
  • Refactor existing codebases to remove accidental complexity so that adding features stops being painful.
  • Help your team adopt AI-assisted coding practices that actually work — from effective use of coding agents to structuring code so these tools can help rather than hinder.
  • Design software with the future in mind: agent-accessible interfaces, clean APIs, and validation layers so your product is ready for a world where AI agents are users too. (More on this in my article Death of the Toolbar.)
  • Work alongside your developers during implementation to make sure the design holds up in practice.
  • Coach your team to think this way themselves, so they don't need me anymore.

The outcome: software that's easy to understand, extend, and maintain — and a team that's equipped to keep it that way.
